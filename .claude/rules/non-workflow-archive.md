# 非 Workflow 变更归档

未走 speckit 流程的变更（运维适配、紧急修复、数据源迁移等），归档方式：

## 规则

1. **不改原始设计文档** — `specs/_archive/{feature}/` 下的 `spec.md`、`plan.md`、`tasks.md`、`data-model.md` 是历史快照，保持原样
2. **在归档目录下新增 `CHANGELOG.md`** — 记录变更原因、范围、影响评估、回滚方案
3. **更新 `specs/_feature-status.md`** — 在对应 feature 下添加变更索引行

## CHANGELOG.md 模板

```markdown
# {feature-name} 变更记录

## YYYY-MM-DD：{变更标题}

**触发原因**：{为什么改}

**变更类型**：运维适配 / 紧急修复 / 数据源迁移 / 其他

**变更人**：{name}

### 变更内容
| 项 | 原方案 | 新方案 |
|----|--------|--------|

### 代码变更
| 文件 | 变更类型 | 说明 |
|------|---------|------|

### 影响评估
- 功能影响
- 性能影响

### 回滚方案
- 如何回滚
```

## 适用场景

- 数据源/API 迁移
- 紧急线上修复
- 配置调整
- 不涉及新 feature 的改动
